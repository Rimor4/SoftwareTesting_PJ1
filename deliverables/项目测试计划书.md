# 项目测试计划书

## 1. 目的

围绕 `SoftwareTestingDemo` 场馆预约管理系统制定本次课程项目测试活动的范围、方法、资源、进度和风险控制策略，为后续测试用例设计、自动化脚本编写和测试总结提供统一依据。

## 2. 测试计划标识

`PJ1-STP-20260411-G01`

## 3. 介绍

被测系统为一个基于 Spring Boot 2.2.2 的体育场馆预约管理系统，包含用户注册登录、场馆浏览、预约下单、留言互动、新闻发布以及后台管理等功能。项目采用 Thymeleaf 作为页面模板，JPA + MySQL 作为持久化方案。

## 4. 测试项

- 被测版本：`SoftwareTestingDemo / 0.0.1-SNAPSHOT`
- 源码路径：`PJ1/SoftwareTestingDemo`
- 测试脚本路径：
  - `src/test/java/com/demo/service/impl/OrderServiceImplTest.java`
  - `src/test/java/com/demo/service/impl/MessageServiceImplTest.java`
  - `src/test/java/com/demo/service/impl/UserServiceImplTest.java`
  - `src/test/java/com/demo/controller/user/UserControllerTest.java`
  - `src/test/java/com/demo/controller/user/OrderControllerTest.java`
  - `src/test/java/com/demo/controller/user/MessageControllerTest.java`
  - `src/test/java/com/demo/controller/AdminControllersTest.java`
  - `src/test/java/com/demo/controller/PublicControllersTest.java`
- 先决条件：Java 8+、Maven 3.8+，Mock 单元/控制器测试不依赖数据库。

## 5. 要测试的特性

- 用户模块：登录、注册、退出、密码校验、资料更新
- 订单模块：新增订单、修改订单、删除订单、订单列表、订单状态流转、按日程查询
- 留言模块：留言发布、修改、删除、前台展示、后台审核
- 后台模块：新闻、用户、场馆、订单的基本管理能力
- 公开页面模块：首页聚合展示、新闻列表、场馆列表、详情页

## 6. 不会被测试的特性

- 真实 MySQL 持久化链路、SQL 兼容性与数据迁移
- 浏览器兼容性、页面布局和复杂前端交互细节
- 文件上传后的真实磁盘写入与静态资源访问，仅验证空文件分支
- 性能、并发压测、安全渗透测试

## 7. 方法

- 单元测试：面向 `service.impl` 中具有状态判断与数据加工逻辑的方法，使用 JUnit 5 + Mockito 进行隔离测试
- 控制器集成测试：面向 `controller` 层功能入口，使用 Mock 请求/响应对象验证请求处理、会话读写、模型填充和跳转行为
- 黑盒技术：等价类划分、边界值分析
- 白盒技术：语句覆盖、判定覆盖
- 完成准则：测试脚本可编译、核心分支均有用例对应、文档与脚本一致、`mvn test` 全部通过

## 8. 测试项通过/失败准则

- 通过：实际结果与预期结果一致，且未出现未处理异常
- 失败：返回值、状态变更、模型属性、会话数据或跳转目标与预期不一致，或出现未预期异常

## 9. 暂停准则和继续准则

- 当基础环境缺失，如 Java、Maven、依赖仓库不可用，导致脚本无法执行时，暂停执行阶段，先完成测试设计和脚本静态审查
- 环境恢复后，从失败的测试类别继续执行，并回归受影响模块

## 10. 测试交付物

## 11. 测试任务

本节列出为完成测试活动所需的全部任务、任务间的依赖关系及所需技能。

### 任务清单与依赖关系

```
需求分析 → 测试设计 → 脚本编写 → 环境搭建 → 测试执行 → 结果分析与总结
                                  ↑
                           （可与脚本编写并行）
```

| 编号 | 任务 | 说明 | 依赖 | 输出 |
| --- | --- | --- | --- | --- |
| T1 | 需求分析 | 阅读课件 PPT、被测系统说明文档及源码，确定测试范围与模块边界 | 无 | 测试范围与模块清单 |
| T2 | 测试设计 | 依据需求分析结果，使用等价类划分、边界值分析、语句/判定覆盖设计测试用例 | T1 | 测试用例设计文档 |
| T3 | 脚本编写 | 将测试用例转化为 JUnit 5 + Mockito 自动化测试脚本 | T2 | `src/test/java` 下 8 个测试类（64 个用例，含 20 个 BVA 用例、4 个权限/业务校验用例、1 个唯一性校验用例） |
| T4 | 环境搭建 | 安装/配置 JDK 17+、Maven 3.8+、本地 Maven 仓库；确认依赖可正常下载 | 无（可与 T3 并行） | 可执行的构建环境 |
| T5 | 测试执行 | 在就绪环境中执行 `mvn test`，收集 Surefire 报告 | T3、T4 | 测试执行报告（通过/失败统计） |
| T6 | 结果分析与总结 | 分析执行结果，记录发现缺陷、风险、改进建议，完成测试总结文档 | T5 | 测试总结文档 |

### 所需技能

| 任务 | 所需技能 |
| --- | --- |
| T1 需求分析 | 阅读理解 Spring Boot 项目结构；了解 MVC 分层架构 |
| T2 测试设计 | 掌握等价类划分、边界值分析、语句/判定覆盖等测试设计方法 |
| T3 脚本编写 | 熟练使用 JUnit 5（`@ExtendWith`、`@Mock`、`@InjectMocks`、断言 API）；掌握 Mockito（`when`/`verify`）；了解 Spring `MockHttpServletRequest`/`MockMultipartFile` |
| T4 环境搭建 | 具备 JDK 安装与 `JAVA_HOME` 配置能力；了解 Maven `pom.xml` 依赖管理与插件配置 |
| T5 测试执行 | 能够执行 Maven 命令行；能够阅读 Surefire 测试报告 |
| T6 结果分析 | 具备缺陷归类与技术文档写作能力 |

## 16. 风险以及应急措施

| 风险 | 影响 | 应急措施 |
| --- | --- | --- |
| 缺少 Java/Maven | 无法执行自动化测试 | 在可用机器安装 JDK 与 Maven 后执行；当前阶段先交付脚本与文档 |
